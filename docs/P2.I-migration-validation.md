# P2.I — Migration Validation Period (ops runbook)

**Status:** in_progress · **Owner:** Bradley · **No code.**

This phase is an *ops process*, not an implementation. Anima's permission-mode
migration requires real-world evidence that `gated` mode is stable enough to
ship as the new install default. This document codifies the validation window,
the telemetry we watch, and the exit criteria.

## Timeline

| Date (UTC) | Event |
|---|---|
| 2026-04-18 | Validation window opens. `gated` mode available as opt-in via Settings UI (P2.H shipped). New installs default to `bypass` (legacy). |
| 2026-05-02 | Earliest exit date (window ≥ 14 days per v14 plan §P2.I). |
| TBD | If exit criteria met → flip install default to `default` (CLI prompts). `bypass` requires explicit setting from that release forward. |
| TBD + 30 days | Second review. If `gated`-mode fallback rate < 1% → flip install default to `gated`. |

## Telemetry source

P7.A (post-launch phase) will ship opt-in telemetry. Until then, the
supervisor (`src-tauri/src/mcp_gate/supervisor.rs`) already tracks the
signal we care about in-memory per session:

- `supervisor_record_gate_crash(session_id)` is called from
  `src/session-lifecycle.js` every time a `gated` spawn's subprocess exits.
- `supervisor_circuit_state(session_id)` reports `{ open, crashes, backoffMs }`
  at any time.
- When the circuit opens, `session-lifecycle.js` downshifts the NEXT spawn
  to `--permission-mode default` and sets `s._spawnMode = 'degraded'`.
  The red banner surfaces to the user.

**Interim measurement (pre-P7.A):** scan `permission_audit.jsonl`
(`~/.local/share/pixel-terminal/permission_audit.jsonl` — P2.F) for gaps
correlated with `s._spawnMode === 'degraded'` banner events in the JS log
(`/tmp/pixel-terminal.log`).

## Exit criteria (v14 plan §P2.I)

All must be true to advance the install default:

1. **≥14 days elapsed** since 2026-04-18.
2. **Gate fallback frequency < 1%** of total sessions, measured across the
   full window. One fallback per session counts once regardless of retry
   behaviour inside that session.
3. **No unresolved crashes** in the supervisor that trace to the gate
   binary itself (`anima_gate`). User-denied tool calls are *not* crashes.
4. **No security regressions.** A new issue in `permission_audit.jsonl`
   showing a tool call bypassing the modal when `gated` was selected
   blocks the default flip.
5. **Manual smoke pass** of the chaos test (v14 §P2.G acceptance #5):
   kill the gate process 3× in 60s → red banner appears, next spawn uses
   `--permission-mode default` (verified via `ps`), audit log shows zero
   new entries after third kill.

## Weekly review cadence

Every Friday during the window, run this checklist:

- [ ] Count gated sessions in the window (count of `s._spawnMode === 'gated'`
      spawn lines in `/tmp/pixel-terminal.log`).
- [ ] Count degraded sessions (spawn lines with `s._spawnMode === 'degraded'`
      or supervisor banner writes).
- [ ] Compute fallback rate = degraded / (gated + degraded).
- [ ] Scan `permission_audit.jsonl` for timeouts or parse errors.
- [ ] Re-run the chaos test if any code touched `src-tauri/src/mcp_gate/`
      during the week.
- [ ] Log the result as a one-line gemini-memory entry:
      `memory_add collection=pixel_terminal type=intake tags=P2.I,validation`
      `source=runbook` document like: `"Week N: gated=X, degraded=Y,
      fallback=Z%. No audit anomalies."` id: `p2i-week-N`.

## Escalation

If at any point during the window the supervisor is seeing >5% fallback:

1. Freeze the install default flip until root cause is identified.
2. Open a tracked issue in gemini-memory with tag `P2.I,blocker`.
3. If root cause is gate binary instability → file a regression bug and
   reopen P2.C/P2.D acceptance.
4. If root cause is CLI drift → run the contract-drift CI workflow
   (`.github/workflows/contract-drift.yml`) and update the parity fixture.

## Connected work

- **P2.F** audit log is the forensic source — keep it.
- **P2.G** supervisor is the runtime signal.
- **P2.H** Settings UI is the opt-in surface.
- **P7.A** telemetry replaces the manual weekly review once it ships.
- **P7.C** weekly post-mortem ritual folds this runbook into the Friday
  digest after launch.

## After exit

1. Open a PR changing the default in `src/app.js` (the fallback in the
   `catch { /* no settings file yet */ }` block) from `bypass` to `default`.
2. Update `docs/architecture.md` to describe the new default.
3. Append a new entry to this runbook documenting the flip date and the
   fallback rate observed over the window.
4. Close Task #20 (P2.I) in the plan tracker.
